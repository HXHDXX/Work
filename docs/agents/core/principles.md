# 核心原则与行为准则

> AGENTS.md 清单的详细说明。

## 核心原则

### 中英双语思维链

- 推理过程可使用中文或英文，不受限制
- 最终交付物（代码注释、文档）遵循项目现有语言习惯
- commit message 使用英文，遵循 Conventional Commits

### 先理解再行动

- 收到任务后，先确认理解正确再动手
- 理解 = 知道要改什么文件、改什么内容、为什么要改
- 不理解就问，不假设
- 静态：优先检索 `./docs`
- 动态：通过构建系统理解

### 验证贯穿始终

- 前置设计验收标准，后置交付验证方案
- 每个交付物必须附带可执行的验证步骤
- 验证方案包括：构建命令、测试命令、手动检查步骤
- 没有验证方案的交付视为未完成

## 行为准则

### 澄清意图再动手

- 需求有歧义就问，不假设用户意图
- 区分「研究/理解」和「实施/修改」意图
- 用户未明确说「实现/修改/创建」时，只做分析和建议

### 深度调研

- 理解需求时，必须输出概念现状（当前代码如何工作）
- 清理技术盲区后再动手，不在盲区中编码

### 简单优先

- 禁止编写超出当前需求的代码与过度抽象（YAGNI）
- 不引入不必要的依赖
- 能用标准库就不用第三方库

### 精准修改

- 聚焦目标，禁止在非相关区域附带重构
- Bug 修复时绝不重构周围代码
- 每次修改有且仅有一个目的

### 验证先行

- 行动前，必须设计并交付可执行的验收标准
- 验收标准必须是可执行的命令或检查步骤
- 修改后必须运行验证，不可跳过

### Trivial 豁免

- 满足以下全部条件时可直接执行，无需完整流程：
  - 修改 ≤ 1 个文件
  - 修改 ≤ 20 行
  - 无架构影响
- 禁止通过连续拆分规避限制
- 且仍必须通过构建验证
- 不满足任一条件时，遵循完整流程（调研 → 计划 → 实施 → 验证）
